# Product Requirements Document (PRD)

## 0. 개요

**제품명**: One Question a Day
**버전**: 1.0 (MVP) → 2.0 (확장)
**목표**: 언어 장벽이 낮고 참여 비용이 극단적으로 낮은 글로벌 마이크로 설문 플랫폼 구축.

---

## 1. 문제 정의

### 1.1 사용자 문제
- 다른 나라 사람들이 "사소한 것들"에 대해 어떻게 생각하는지 궁금하다.
- 설문/조사에 참여하기엔 시간과 노력이 든다.
- 문화 차이를 데이터로 체감할 수 있는 플랫폼이 없다.

### 1.2 기회
- 10초 만에 참여 가능한 인터랙션 = 낮은 장벽
- 결과를 시각적으로 즉시 확인 = 즉각적 보상
- 공유 가능한 콘텐츠 = 바이럴 루프
- 시간이 쌓일수록 데이터 자체가 자산

---

## 2. 페르소나

### 2.1 Casual Curious (주 사용자, 70%)
- 20–35세, 모바일 중심, 평균 세션 1분
- 트위터/인스타에서 "이런 거 해봤음?" 콘텐츠 즐김
- 매일 오지는 않지만, 흥미로운 질문은 친구에게 공유
- **니즈**: 재미, 짧은 시간, 공유할 거리

### 2.2 Daily Ritualist (충성 사용자, 20%)
- 25–45세, 데스크탑+모바일
- "오늘의 퀴즈" 같은 데일리 루틴을 좋아함
- 매일 알림 받고 답함. streak 유지 욕구
- **니즈**: 일관성, 개인 통계, 지속 가능한 루틴

### 2.3 Data Lover (소수 핵심 옹호자, 10%)
- 30–50세, 저널리스트/연구자/블로거
- 아카이브를 파서 글과 인사이트 생산
- **니즈**: 데이터 접근성, 신뢰성, 인용 가능성

## 3. 핵심 사용자 플로우

### 3.1 신규 방문자 플로우
```
1. 랜딩 (/)
   ↳ 오늘의 질문 + 선택지만 보임. 결과 영역은 블러 처리.
2. 선택지 클릭 → 애니메이션 → 결과 공개
   ↳ 내 답 강조
   ↳ 우리나라 비율
   ↳ 세계지도 (국가별 최다 선택)
   ↳ 전체 바 차트
3. 공유 버튼 → OG 이미지 포함 링크 복사/SNS
4. "Subscribe for daily reminders" (이메일, 선택적)
5. "See past questions" → /archive
```

### 3.2 재방문자 플로우
```
1. / 접근 → 쿠키로 이미 답했는지 확인
2a. 답했으면 → 결과 화면 바로
2b. 안 했으면 → 오늘의 질문 노출
3. 어제 질문 요약 배너 (선택적 표시)
```

### 3.3 질문 제안 플로우
```
1. /suggest 진입
2. 질문 텍스트 + 선택지 2~4개 + 언어 선택
3. 제출 → 관리자 큐로 이동
4. 승인 시 이메일 알림 (제공 시)
```

## 4. 기능 요구사항

### 4.1 MVP (v1.0) — P0

#### F-01: 오늘의 질문 표시
- **AC-1**: UTC 00:00 기준 "오늘"의 질문 하나가 메인에 표시됨
- **AC-2**: 아직 publish 안 된 날이면 폴백 질문 또는 카운트다운
- **AC-3**: 질문 텍스트는 사용자 로케일로 번역되어 표시, 번역 없으면 영어 폴백

#### F-02: 응답 제출
- **AC-1**: 선택지 클릭 시 500ms 내 응답 반영
- **AC-2**: 같은 세션 해시 + 같은 질문 조합은 두 번째 요청 시 409 Conflict
- **AC-3**: 응답과 동시에 `country_code`가 자동 기록됨 (Cloudflare 헤더 기반)

#### F-03: 결과 공개
- **AC-1**: 응답 완료한 사용자만 결과 API 접근 가능
- **AC-2**: 전체 통계(바 차트) + 우리나라 통계 + 세계지도 표시
- **AC-3**: 내가 선택한 옵션이 시각적으로 강조됨

#### F-04: 세계지도 시각화
- **AC-1**: 국가별 "가장 많이 선택된 답"을 색상으로 구분
- **AC-2**: 국가 hover 시 툴팁으로 국가명, 참여자 수, 옵션별 비율 표시
- **AC-3**: 참여자 10명 미만 국가는 회색(통계 불충분) 표시
- **AC-4**: 모바일에서 터치 인터랙션 작동

#### F-05: 공유
- **AC-1**: "공유" 버튼 클릭 시 트위터/카카오톡/링크복사 옵션
- **AC-2**: 공유되는 링크는 `/share/:questionId?opt=xxx` 형태로 OG 이미지 포함
- **AC-3**: OG 이미지는 질문 텍스트 + 주요 결과를 담음

#### F-06: 중복 응답 방지
- **AC-1**: localStorage + 세션 쿠키 + DB UNIQUE 제약의 3중 방어
- **AC-2**: 쿠키 지워도 같은 IP+UA 조합은 차단 (최소 1시간)
- **AC-3**: 명백한 봇 패턴은 차단 ([`docs/ARCHITECTURE.md`](./ARCHITECTURE.md) 참조)

---

### 4.2 Phase 2 (v1.1) — P1

#### F-07: 아카이브
- **AC-1**: `/archive`에서 지난 질문 리스트 무한 스크롤
- **AC-2**: 날짜/카테고리 필터
- **AC-3**: 각 항목 클릭 시 `/archive/:date` 상세 (해당 날의 결과 전체)

#### F-08: 다국어
- **AC-1**: 언어 감지: URL 세그먼트 `/[locale]` > `Accept-Language` 헤더 > 기본 en
- **AC-2**: 최소 en, ko 지원. 이후 ja, es, zh, fr, de 순 추가
- **AC-3**: 질문 번역이 없는 언어는 영어 폴백, 작은 "Translate" 버튼 노출

#### F-09: 이메일 알림
- **AC-1**: 이메일 입력 시 옵트인
- **AC-2**: 매일 UTC 00:00 + 3h 기준 일괄 발송
- **AC-3**: 언서브스크라이브 링크 모든 메일 하단

#### F-10: 질문 제안
- **AC-1**: 비로그인으로 제안 가능, 1 IP당 1일 3건 제한
- **AC-2**: 욕설/스팸 1차 필터링 (단순 워드 리스트 + 길이 검증)
- **AC-3**: 관리자 페이지에서 승인/거절/스케줄링

## 5. 비기능 요구사항

### 5.1 성능
- LCP < 2.5s (모바일 3G)
- 응답 제출 API p95 < 300ms
- Lighthouse Performance 90+

### 5.2 접근성
- WCAG 2.1 AA 준수
- 키보드 내비게이션 전체 지원
- 색각 이상자 대비 (색 + 패턴 병행)
- 스크린 리더 지원

### 5.3 보안/프라이버시
- 개인정보 최소 수집 (이메일만, 선택적)
- IP는 해시 후 저장, 원문 저장 금지
- GDPR/K-GDPR 준수 (쿠키 동의, 삭제 요청 대응)

### 5.4 국제화
- 모든 UI 문자열은 메시지 파일 통해서만
- RTL 언어 대비 구조 (CSS logical properties 사용)
- 날짜/숫자 포매팅은 `Intl` API

### 5.5 신뢰성
- 업타임 목표 99.5%
- 하루에 질문이 공개되지 않는 사태는 **P0 장애**
- 최소 30일치 질문 버퍼 유지
